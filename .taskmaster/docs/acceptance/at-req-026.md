## E. Write authority (detect-and-revert)

- **AT-026.16 (P0)** — Given the volunteer (or their agent), When they read, comment, and pull, Then all succeed; When they change a PM-TREE status by hand, Then the change is detected and reverted with an explanatory comment on the item and a low-tone notification to the volunteer — detect-and-revert, because vendor permissions cannot block status-only; and When they change a DEV-TREE status by hand or its vendor automation fires, Then it STANDS untouched — the dev tree is revert-EXEMPT (positive probe). [d82] [cross: AT-008.13]
- **AT-026.17 (P0)** — Given a pulled PM requirement whose linked dev-tree work is merged and whose acceptance is evidenced, When the verified completion is recorded against the pull, Then the PM item moves to Done — the ONLY path to PM Done (a PM Done not backed by a verified completion never survives; authorized platform cancellations are a separate lifecycle authority, AT-026.18/20); Given the negative set — a PM Done attempted with a linked dev item still OPEN, with acceptance evidence MISSING, with evidence recorded against ANOTHER pull, with dev leaves manually marked Done WITHOUT merges (manual dev edits stand, but never authorize PM Done), and a vendor automation touching a PM item (five negatives), Then no PM item advances and detect-and-revert corrects the state; and Given a DEV leaf's PR merging via the GitHub integration, Then that leaf flips Done vendor-natively and is NOT reverted — the two trees' done-paths are distinct and both proven. [d82: verified completion replaces the task-level matching-merge formula at the PM tier; the merge formula lives on at the dev tier as vendor mechanics] [cross: AT-008.13/AT-012.05]
- **AT-026.18 (P0)** — Given platform-only operations, When exercised, Then the platform can seed PM items, push the dev decomposition, invite, revert (PM tree only), and cancel; the volunteer CAN create a scope-addition linked PM item [cross: REQ-025.02] but cannot cancel or seed PM items; the NGO can do none; and a platform-originated SCOPE-ADDITION item cannot arise — platform creation is limited to seeding/decomposition/lifecycle cases (v1 scope-addition items are volunteer-created, exclusively).
