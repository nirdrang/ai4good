#### REQ-036: Dev-Authored Project PRD & Completion Gate

**Description:** Discovery output is a scope contract, not a task source. Alongside the scope-doc requirement items, kickoff seeds the PM tree with one bootstrap item ("Author the project PRD") — a pullable PM-tree requirement: the volunteer pulls it, and authoring the project PRD in the repo from the Discovery scope is metered, fuel-billed, and attributed to that pulled item (requirement-level, REQ-034). Clarifying blockers (REQ-024) are expected and flow to the project Q&A log (REQ-010). When the volunteer marks the PRD ready, an automated scorer compares it to the Discovery scope, producing a completion score and named gaps. A score at or above the threshold decomposes the build backlog from the PRD onto the DEV TREE (REQ-026) and constitutes the bootstrap item's verified-completion evidence — the gate pass moves the bootstrap to Done through the authorized completion path; below the threshold, the bootstrap stays in progress and the volunteer iterates on the gaps and re-scores — bounded by project fuel, not an attempt cap.

**Acceptance criteria:**
- [ ] Kickoff seeds the bootstrap item on the PM tree (with the scope requirement items, REQ-026); the dev tree is empty before the gate passes.
- [ ] The PRD lives in the repo; the volunteer pulls the bootstrap item and authoring is metered and attributed to that pulled PM item (REQ-009/REQ-034).
- [ ] Clarifying questions ride the blocker flow (REQ-024) to the Q&A log (REQ-010).
- [ ] The scorer evaluates Discovery-scope coverage (stories, acceptance criteria, data-sensitivity handling, constraints), producing a score and gaps, and runs fuel-metered.
- [ ] Gate: at or above the threshold the tree is decomposed and pushed (REQ-026); below, a gap report is produced and the PRD phase continues. **[DECISION: OD-7 — threshold + scorer configuration; pilot-tuned.]**
- [ ] The NGO sees PRD-phase status on the project page (the PM tree with the bootstrap item and score state — never any dev-tree content); the NGO is not an approver — the scorer is the gate and NGO input is via clarifications.
- [ ] Score events are recorded and notified (the volunteer gets the gap report; the NGO gets gate-pass and backlog-live — REQ-016).

**Dependencies:** REQ-004, REQ-024, REQ-026, REQ-034, REQ-009.
